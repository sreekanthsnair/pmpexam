PMP Quick Summary 2025: Fast-Track Reference

This summary covers key PMP topics for the 2025 exam, based on PMI's Exam Content Outline (ECO). Focus: People (42%), Process (50%), Business Environment (8%). Includes predictive, agile, and hybrid essentials.

1. People Domain (42% of Exam)

Focus: Leadership, team management, stakeholder engagement.

  • Leadership Styles: Directive (new teams), Servant (agile), Transformational (change-driven).
  • Conflict Resolution: Collaborate (win-win), Compromise (give-and-take), Avoid (delay), Force (last resort).
  • Team Building: Tuckman’s stages - Forming, Storming, Norming, Performing, Adjourning.
  • Motivation: Recognize efforts, provide support (Maslow, Herzberg).
  • Stakeholder Engagement: Identify, analyze (power/interest grid), tailor communication.
  • Agile Team: Self-organizing, empowered, cross-functional.
  • Key Tools: RACI chart (roles), comm plan, stakeholder register.

Example: Team misses deadlines - First discuss performance, then train (ECO Task 5).

2. Process Domain (50% of Exam)

Focus: Project execution, planning, monitoring, control.

  • Process Groups: Initiating, Planning, Executing, Monitoring & Controlling, Closing (5 total).
  • Scope: Define via charter, WBS (work breakdown structure), control scope creep.
  • Schedule: Critical Path Method (CPM) - longest path, zero float. Crashing (add resources, ↑ cost), Fast-tracking (overlap tasks, ↑ risk).
  • Cost: Budget = cost baseline + reserves (contingency: known unknowns, management: unknown unknowns).
  • EVM Formulas:
    • CV = EV - AC (Cost Variance, + = under budget)
    • SV = EV - PV (Schedule Variance, + = ahead)
    • CPI = EV / AC (Cost Performance Index, >1 = good)
    • SPI = EV / PV (Schedule Performance Index, >1 = good)
    • EAC = BAC / CPI (if current trend continues)
    • TCPI = (BAC - EV) / (BAC - AC) (to complete on budget)
  • Risk: Identify (register), analyze (probability × impact = EMV), respond (avoid, mitigate, transfer, accept).
  • Quality: QA (process improvement), QC (inspect deliverables). Tools: Pareto (80/20), Ishikawa (cause-effect), Control Charts (trends).
  • Procurement: Fixed-price (clear scope, seller risk), Cost-plus (flexible, buyer risk), T&M (moderate risk).
  • Change Control: Submit change request → CCB (Change Control Board) → update plans.
  • Agile Process: Sprints (1-4 weeks), Daily Scrum (15 min), backlog prioritized by Product Owner, retrospectives for improvement.

Example: EV = $200K, AC = $250K, CPI = 0.8 (over budget), ECO Task 9.

3. Business Environment Domain (8% of Exam)

Focus: Strategic alignment, organizational impact.

  • Alignment: Ensure project supports org goals - assess changes if strategy shifts.
  • Benefits: Define value (ROI, customer satisfaction), track realization post-project.
  • Compliance: Follow legal, ethical standards (PMI Code of Ethics).
  • External Factors: Market shifts, regulations - adapt via change requests.
  • Key Tools: Business case, benefits management plan.

Example: Market changes project need - Assess impact, submit change (ECO Task 1).

4. Agile and Hybrid Essentials (50% of 2025 Exam)

Focus: Iterative delivery, adaptability.

  • Agile Principles: Working product > documentation, customer collaboration, respond to change (Agile Manifesto).
  • Roles: Product Owner (backlog), Scrum Master (process), Team (self-organizing).
  • Events: Sprint Planning, Daily Scrum, Sprint Review (demo), Retrospective (improve).
  • Artifacts: Product Backlog, Sprint Backlog, Increment.
  • Velocity: Measure of work completed per sprint - adjust scope, not time.
  • Hybrid: Predictive (fixed scope) + Agile (iterative delivery). Example: Design predictive, dev agile.
  • Key Tools: Kanban board, burndown chart.

Example: Mid-sprint change? Defer to backlog, protect sprint goal.

5. Quick Tips for 2025 PMP Exam
  • Question Types: Situational (60-70%), formulas (5-10%), tools (10-15%), ethics (5-10%).
  • Strategy: Eliminate distractors, think PMI way (process over instinct).
  • Focus Areas: EVM, risk, agile, stakeholder mgmt, change control.
  • Practice: Use simulators, review ECO tasks daily.
